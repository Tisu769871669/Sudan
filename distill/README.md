# 苏丹客服蒸馏资产

这组资产基于 `mem/wx_pad_private_message.xlsx` 中的人工客服聊天记录提炼而来，用于增强当前 `main + FAQ + bridge` 架构。

目标不是替代 FAQ 或业务规则，而是把真人客服的：

- 关系感
- 口语感
- 承接方式
- 情绪处理
- 交接习惯

抽成可复用的 prompt / persona 资产。

## 设计原则

- 让蒸馏结果负责“怎么说”
- 让知识库和规则负责“说什么不能错”
- 保留真人感，但收住高风险表达
- 优先适配当前已上线的 `main` agent 与 `agent-bridge`

## 文件说明

- `sudan-chat-distillation.md`
  - 对聊天记录的结构分析、主要发现、建议边界
- `sudan-service-heuristics.md`
  - 可直接复用到 agent 的风格、启发式和反模式

## 主要结论

- 真人客服回复明显偏短，平均长度约 31 字，中位数约 28 字
- 强项不是长篇解释，而是：
  - 快速接话
  - 情绪安抚
  - 关系维护
  - 交接售后
- 高频语言信号：
  - `亲爱的`
  - `谢谢您`
  - `爱您`
  - `拥抱`
  - `玫瑰`
- 对业务问题的典型结构：
  - 先接情绪 / 先确认
  - 再给动作
  - 最后交接或补一句关怀

## 风险提醒

原始聊天里存在少量不适合直接复刻的内容，例如：

- 过度亲密化表达
- 对身体改善的强归因
- 过强的承诺式说法

这些在接入当前客服 agent 时已经做了收敛处理。
